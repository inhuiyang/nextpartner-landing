# Claude Code 작업 지시서: 넥스트파트너 랜딩페이지 생성/고도화

## 0. 작업 목적

넥스트파트너 랜딩페이지를 아래 7개 핵심 섹션 중심으로 생성/고도화한다.

1. Hero - 영상 + 슬로건
2. 메인 타이틀과 브랜드 스토리
3. AI 기반 해결 방식 소개
4. 이용 프로세스
5. 기업 채용 배너
6. 신뢰 지표 / 운영 강점
7. Final CTA

이번 랜딩페이지는 `카카오모빌리티 홈페이지(https://www.kakaomobility.com/)`를 레퍼런스로 삼아, 큰 타이포그래피, 명확한 슬로건, 넓은 여백, 스토리 중심 섹션, 카드형 정보 구조를 활용한다.

단, 카카오모빌리티의 디자인을 그대로 복제하지 말고, 넥스트파트너의 서비스 성격에 맞게 재해석한다.

---

## 1. 작업 전 필수 확인 사항

작업 시작 전 반드시 아래 내용을 확인한다.

1. 프로젝트 루트의 `CLAUDE.md` 또는 `claude.md` 파일을 먼저 읽는다.
2. 기존 프로젝트의 프레임워크, 폴더 구조, 코드 컨벤션, 라우팅 방식을 확인한다.
3. 기존에 구현된 공통 컴포넌트가 있다면 구조만 참고하되, 디자인시스템이 없으므로 필요한 스타일은 해당 페이지 안에서 정리한다.
4. 첨부된 기존 넥스트파트너 랜딩페이지 이미지를 참고하되, 컬러와 톤은 코랄/오렌지 계열로 전환한다.
5. 실제 영상 파일이 있다면 지정 경로를 사용하고, 없다면 영상형 placeholder를 만든다.

### 주의사항

- 디자인시스템이 없으므로 “기존 디자인시스템 재사용” 관련 지시는 적용하지 않는다.
- 임의로 과도하게 많은 컬러를 만들지 않는다.
- 주요 컬러는 코랄/오렌지 계열로 통일한다.
- `데이터 준비 중`, `Coming soon`처럼 미완성으로 보이는 문구는 사용하지 않는다.
- 실제 수치 데이터가 없는 경우 임의 수치를 만들지 않는다.
- `업계 1위`, `검증 완료`, `최고`처럼 근거 없는 과장 표현을 사용하지 않는다.
- 액티브 시니어 사용자를 고려해 텍스트 크기, 대비, 버튼 클릭 영역을 충분히 확보한다.
- 불필요한 신규 라이브러리는 설치하지 않는다.

```text

```

### 참고할 요소

카카오모빌리티 홈페이지에서 아래 특징을 참고한다.

1. 짧고 강한 슬로건 중심의 Hero 구성
2. 넓은 여백과 큰 타이포그래피
3. 브랜드 철학을 먼저 보여주는 스토리텔링 구조
4. 서비스와 기술을 카드 또는 섹션 단위로 나누는 방식
5. 이미지/영상 비주얼을 활용한 몰입감 있는 첫 화면
6. 하단 CTA까지 자연스럽게 이어지는 긴 호흡의 랜딩 구성

### 적용 방식

넥스트파트너에는 아래처럼 재해석한다.

- `we move life`처럼 짧고 인상적인 슬로건을 사용하되, 넥스트파트너는 “경험과 기회를 연결한다”는 메시지로 전환한다.
- 이동/모빌리티가 아니라 “시니어의 경험 이동”, “일자리 기회 연결”, “AI 기반 채용 연결”을 중심으로 표현한다.
- 카카오모빌리티의 블랙/화이트 기반 무드는 그대로 따르지 않고, 코랄/오렌지 계열의 따뜻하고 활력 있는 톤으로 구성한다.
- 기술 기업의 세련됨과 액티브 시니어 서비스의 따뜻함이 함께 느껴지도록 한다.

### 컬러 방향

코랄/오렌지 계열을 메인으로 사용한다.

권장 컬러 예시:

```css
:root {
  --color-primary: #FF6B4A;
  --color-primary-dark: #E95735;
  --color-primary-light: #FFF0EA;
  --color-coral: #FF7A66;
  --color-orange: #FF9A3D;
  --color-cream: #FFF8F3;
  --color-text-primary: #1F1F1F;
  --color-text-secondary: #5F5F5F;
  --color-border: #F1DDD4;
  --color-white: #FFFFFF;
}
```

### 사용 기준

- Primary CTA: 코랄 또는 오렌지
- Secondary CTA: 화이트 배경 + 코랄 텍스트 + 코랄 라인
- 배경: 크림/오프화이트 계열
- 강조 카드: 코랄/오렌지 그라데이션
- 본문 텍스트: 진한 차콜
- 보조 텍스트: 회색 계열
- 전체적으로 너무 붉거나 자극적으로 보이지 않도록 크림 배경과 충분한 여백을 사용한다.

## 5-2. Hero - 영상 + 슬로건

### 섹션 목적

첫 화면에서 넥스트파트너의 정체성을 강하게 전달한다.  
카카오모빌리티의 큰 슬로건형 Hero를 참고해, 짧고 기억하기 쉬운 문장과 영상형 비주얼을 중심으로 구성한다.

### 레이아웃

Desktop 기준:

- 좌측: 라벨, 슬로건, 설명, CTA
- 우측: 영상 또는 영상형 비주얼 카드
- 배경: 크림 배경 + 코랄/오렌지 그라데이션 포인트
- 콘텐츠 최대 폭: 1212px 내외
- Hero 높이: 첫 화면에서 충분히 존재감 있게 보이도록 구성

Mobile 기준:

- 상단: 라벨 + 슬로건
- 중단: 설명 + CTA
- 하단: 영상 카드
- 전체 1열 구조

### Hero 영상 영역

실제 영상 파일이 있는 경우:

```text
/video/nextpartner_intro
```

- 위 경로의 영상을 사용한다.
- `<video>` 태그 또는 프로젝트에서 사용하는 미디어 컴포넌트를 사용한다.
- 자동재생을 적용한다.
- 자동재생 시 `muted`, `playsInline`, `loop` 속성을 함께 적용한다.
- 접근성을 위해 영상 설명 텍스트 또는 `aria-label`을 제공한다.

실제 영상 파일이 없는 경우:

- 영상 플레이어 형태의 placeholder UI를 만든다.
- 코랄/오렌지 그라데이션 배경 위에 play 버튼을 배치한다.
- 서비스 화면 미리보기 카드, AI 기능 태그, 인물/일자리 연결을 연상시키는 그래픽을 구성한다.
- 단순 회색 박스처럼 보이지 않도록 한다.

## 5-3. 메인 타이틀과 브랜드 스토리

### 섹션 목적

넥스트파트너가 단순 구인구직 사이트가 아니라, 시니어의 경험이 다시 사회와 기업 안에서 활용될 수 있도록 연결하는 브랜드임을 전달한다.

### 섹션 타이틀

```text
시니어의 경험이
다음 기회로 이어지도록
```

### 서브 타이틀

```text
일하고 싶은 시니어와 좋은 인재가 필요한 기업 사이에는
아직 충분히 연결되지 못한 가능성이 있습니다.
```

### 브랜드 스토리 본문

```text
넥스트파트너는 50+ 액티브 시니어가 가진 경험과 태도, 일에 대한 책임감이
다시 의미 있는 기회로 이어질 수 있도록 돕는 플랫폼입니다.

구직자에게는 자기소개서 작성, 사진 준비, 면접 대비처럼 부담스러운 과정을
AI 기능으로 더 쉽게 시작할 수 있도록 돕고,
기업에게는 필요한 직무에 맞는 시니어 인재를 효율적으로 검토할 수 있는
채용 연결 흐름을 제공합니다.

우리는 단순히 공고를 보여주는 것이 아니라,
사람의 경험을 이해하고 그 경험이 필요한 자리와 연결되는 방식을 만듭니다.
```

### 브랜드 키워드 카드

3개 카드로 구성한다.

#### 1. 경험

```text
오랜 시간 쌓아온 경력과 태도를 새로운 일의 기회로 연결합니다.
```

#### 2. 준비

```text
자기소개서, 사진, 복장, 면접 준비를 AI로 더 쉽게 시작할 수 있도록 돕습니다.
```

#### 3. 연결

```text
구직자와 기업이 서로에게 맞는 조건으로 만날 수 있는 흐름을 제공합니다.
```

### UI 요구사항

- 카카오모빌리티의 브랜드 소개 섹션처럼 스토리 카드와 큰 타이틀을 조합한다.
- 좌측에는 큰 타이틀과 본문, 우측에는 키워드 카드 3개를 배치한다.
- 카드에는 코랄/오렌지 포인트 아이콘 또는 숫자 배지를 사용한다.
- 모바일에서는 1열로 정리한다.

## 5-4. AI 기반 해결 방식 소개

### 섹션 목적

AI 기능을 서비스의 핵심 차별점으로 보여준다.  
기능명과 설명이 서로 맞도록 구성하고, 각 기능이 어떤 구직/채용 문제를 해결하는지 명확히 표현한다.

### 섹션 타이틀

```text
AI가 구직 준비의 막막함을
하나씩 줄여드립니다
```

### 섹션 설명

```text
넥스트파트너의 AI 기능은 자기소개서 작성, 사진 준비, 면접 연습, 기업 이해까지
구직자가 실제 지원 과정에서 어려워하는 단계를 중심으로 설계됩니다.
```

### 기능 카드 구성

#### 1. AI 자기소개서

설명:

```text
간단한 질문에 답하면 경력과 강점을 바탕으로 자기소개서 초안을 생성합니다.
```

기대 효과:

```text
작성 부담 감소
경력 정리 시간 단축
지원서 완성도 향상
```

#### 2. AI 증명사진 생성

설명:

```text
지원 서류에 필요한 프로필 사진을 간편하게 준비할 수 있도록 돕습니다.
```

기대 효과:

```text
사진 준비 부담 감소
비대면 준비 가능
지원 서류 완성도 향상
```

#### 3. AI 복장 점검

설명:

```text
면접이나 프로필 촬영 전, 지원 직무와 상황에 어울리는 복장인지 점검할 수 있도록 돕습니다.
```

기대 효과:

```text
첫인상 준비
면접 불안 감소
상황에 맞는 이미지 점검
```

#### 4. AI 모의면접

설명:

```text
예상 질문을 바탕으로 답변을 연습하고, 면접 전 말할 내용을 정리할 수 있도록 돕습니다.
```

기대 효과:

```text
면접 연습
답변 구조화
자신감 향상
```

#### 5. AI 기업브리핑

설명:

```text
지원하려는 기업과 직무 정보를 쉽게 이해할 수 있도록 핵심 내용을 요약해 제공합니다.
```

기대 효과:

```text
기업 이해도 향상
지원 전 정보 탐색
면접 준비 보완
```

### UI 요구사항

- 기능 카드는 5개를 사용한다.
- Desktop에서는 3+2 그리드 또는 가로 스크롤 없는 카드 그리드를 사용한다.
- Tablet에서는 2열, Mobile에서는 1열로 배치한다.
- 각 카드에는 아이콘, 기능명, 설명, 기대 효과 태그를 포함한다.
- Hover 시 카드가 살짝 떠오르거나 border가 코랄 컬러로 강조되도록 한다.
- `AI 증명사진 생성` 카드에 일자리 추천 설명이 들어가는 식의 불일치를 절대 만들지 않는다.

## 5-5. 이용 프로세스

### 섹션 목적

구직자와 기업이 넥스트파트너를 어떻게 이용하는지 단계별로 이해할 수 있도록 한다.  
AI 기능과 자연스럽게 연결되는 흐름으로 수정한다.

### 섹션 타이틀

```text
구직 준비부터 채용 연결까지
단계별로 쉽게 진행합니다
```

### 섹션 설명

```text
구직자는 AI 기능을 활용해 지원 준비를 시작하고,
기업은 필요한 조건에 맞는 시니어 인재를 효율적으로 검토할 수 있습니다.
```

### 탭

```text
구직자
기업
```

### 구직자 프로세스

#### 01. 기본 정보 입력

```text
희망 직무, 근무 지역, 경력, 지원 목적을 간단히 입력합니다.
```

#### 02. AI 구직 준비

```text
AI 자기소개서, 증명사진, 복장 점검, 모의면접 기능으로 지원 준비를 시작합니다.
```

#### 03. 기업 정보 확인

```text
AI 기업브리핑을 통해 지원하려는 기업과 직무를 쉽게 이해합니다.
```

#### 04. 지원 및 연결

```text
준비한 내용을 바탕으로 관심 있는 공고에 지원하고 기업과 연결됩니다.
```

### 기업 프로세스

#### 01. 채용 정보 등록

```text
필요한 직무, 근무 조건, 우대 경험을 등록합니다.
```

#### 02. 지원자 정보 확인

```text
지원자의 경력, 자기소개서, 준비 상태를 한눈에 확인합니다.
```

#### 03. 적합 인재 검토

```text
직무 조건과 지원자 정보를 바탕으로 적합한 시니어 인재를 검토합니다.
```

#### 04. 면접 및 채용 진행

```text
적합한 인재에게 면접을 제안하고 채용 과정을 진행합니다.
```

### UI 요구사항

- Desktop: 4단계 가로 스텝 카드
- Mobile: 세로 타임라인
- 구직자/기업 탭 전환 기능 구현
- 선택된 탭은 코랄/오렌지 배경 또는 라인으로 명확히 강조
- 각 단계에는 숫자 배지, 제목, 설명을 포함한다.
- 프로세스 설명은 너무 길지 않게 카드당 2줄 내외로 유지한다.

```text
넥스트파트너가 기업의 채용 과정을 더 효율적으로 돕습니다.
```

Primary:

```text
기업 채용 문의하기
```

Secondary:

```text
시니어 인재풀

AI 기반 정보 정리
면접 제안
```

### UI 요구사항

- 코랄/오렌지 그라데이션 가로 배너로 구성한다.
- 배너 내부에 카피, 태그, CTA 버튼을 배치한다.
- Desktop에서는 텍스트 좌측, CTA 우측 배치.
- Mobile에서는 텍스트와 버튼을 세로 정렬한다.
- 배너는 페이지 중간에서 명확한 전환 포인트가 되어야 한다.

## 5-7. 신뢰 지표 / 운영 강점

### 섹션 목적

실제 수치가 없는 상태에서도 서비스가 어떤 기준과 방향으로 운영되는지 전달한다.  
정량 지표가 아닌 운영 강점 중심으로 구성한다.

### 섹션 타이틀

```text
구직자와 기업 모두에게
안심할 수 있는 연결 경험을 제공합니다
```

### 섹션 설명

```text
넥스트파트너는 액티브 시니어의 구직 여정과 기업의 채용 흐름을 함께 고려해
더 쉽고 명확한 연결 경험을 설계합니다.
```

### 운영 강점 카드

#### 1. 50+ 맞춤 구직 흐름

```text
액티브 시니어가 복잡한 절차에 막히지 않도록 간결한 정보 구조와 큰 CTA를 제공합니다.
```

#### 2. AI 기반 지원 준비

```text
자기소개서, 사진, 복장, 면접, 기업 이해까지 실제 지원 과정에 필요한 준비를 돕습니다.
```

#### 3. 기업 채용 검토 지원

```text
기업이 지원자 정보를 빠르게 이해하고 검토할 수 있도록 채용 흐름을 정리합니다.
```

#### 4. 구직자·기업 연결 구조

```text
구직자에게는 더 쉬운 지원 경험을, 기업에게는 더 효율적인 인재 검토 경험을 제공합니다.
```

### 보조 정보 영역

실제 숫자가 없는 경우 숫자형 지표 대신 아래와 같은 문구형 지표를 사용한다.

```text
구직 준비 지원
기업 정보 이해
면접 대비
채용 연결
```

실제 수치가 제공될 경우에만 아래 형식으로 교체한다.

```text
누적 구직자 수 000명
등록 기업 수 000개
진행 공고 수 000건
채용 연결 사례 000건
```

### UI 요구사항

- 4개 강점 카드를 2x2 또는 4열 그리드로 배치한다.
- 카드 상단에는 짧은 라벨 또는 아이콘을 둔다.
- 숫자가 없는 상황에서 `000명`, `000건` placeholder를 화면에 노출하지 않는다.
- 숫자 데이터가 없다면 문구형 지표만 사용한다.
- 배경은 크림 또는 아주 연한 코랄 톤을 사용해 안정감 있게 구성한다.

## 6. 반응형 기준

### Desktop

- 기준 화면: 1440px
- 콘텐츠 최대 폭: 1212px 내외
- Hero는 2열 구조
- AI 기능 카드는 3+2 또는 균형 잡힌 그리드
- 프로세스는 4단계 가로 스텝
- 주요 섹션 간 여백은 충분히 확보한다.

### Tablet

- Hero는 필요 시 1열 또는 2열로 자연스럽게 전환한다.
- 카드 섹션은 2열 그리드 중심으로 구성한다.
- CTA 버튼은 콘텐츠 폭에 맞춰 배치한다.

### Mobile

- 전체 1열 구조
- Hero 영상은 텍스트 아래 배치
- CTA 버튼은 세로 배치
- AI 기능 카드는 1열
- 프로세스는 세로 타임라인
- 카드 클릭 영역과 버튼 높이를 충분히 확보한다.
- Header는 모바일 메뉴 또는 간소화된 내비게이션을 사용한다.

## 7. 접근성 / 사용성 기준

- 본문 폰트는 최소 16px 이상 사용한다.
- 주요 설명 텍스트는 17~18px 수준을 권장한다.
- 버튼 높이는 최소 48px 이상 확보한다.
- CTA 버튼과 카드 클릭 영역은 터치하기 쉽게 구성한다.
- 연한 회색 텍스트 사용을 최소화하고 충분한 대비를 확보한다.
- 영상에는 대체 설명 또는 접근 가능한 라벨을 제공한다.
- 키보드 포커스 상태가 시각적으로 확인되어야 한다.
- 탭 인터랙션을 구현할 경우 키보드 접근성과 ARIA 속성을 고려한다.

## 9. 완료 기준

아래 조건을 모두 충족해야 한다.

- 디자인시스템 관련 내용 없이 작업했다.
- 카카오모빌리티 홈페이지를 레퍼런스로 참고하되 그대로 복제하지 않았다.
- 전체 컬러 톤이 코랄/오렌지 계열로 구성되어 있다.
- 랜딩페이지가 아래 7개 핵심 섹션 중심으로 구성되어 있다.
  - Hero - 영상 + 슬로건
  - 메인 타이틀과 브랜드 스토리
  - AI 기반 해결 방식 소개
  - 이용 프로세스
  - 기업 채용 배너
  - 신뢰 지표 / 운영 강점
  - Final CTA
- Hero 영역에서 영상 또는 영상형 placeholder가 제공된다.
- 서비스 슬로건이 첫 화면에서 명확하게 보인다.
- 브랜드 스토리가 별도 섹션으로 구성되어 있다.
- AI 기능명과 설명이 서로 자연스럽게 맞는다.
- AI 기능은 문제 해결 방식으로 설명되어 있다.
- 구직자/기업 이용 프로세스가 탭으로 구분되어 있다.
- 기업 채용 배너가 중간 CTA 역할을 한다.
- 실제 수치가 없는 경우 숫자형 placeholder를 노출하지 않는다.
- 근거 없는 수치나 과장 표현을 사용하지 않았다.
- 모바일에서 모든 섹션이 자연스럽게 1열로 정리된다.
- 버튼, 카드, 텍스트 크기가 액티브 시니어 사용자가 읽고 누르기 쉽다.
- 빌드 오류와 타입 오류가 없어야 한다.

## 11. 최종 요청
